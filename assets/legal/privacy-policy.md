# Privacy Policy

**Effective Date: 2025-11-25**

---

## 1. Introduction

Peek4c (the "Application", "we") respects and protects user privacy. This Privacy Policy explains how we collect, use, store, and protect your information.

IMPORTANT STATEMENT: This Application is completely localized, with all data stored only on your device. We do not upload any data to remote servers.

## 2. Information Collection

### 2.1 Information We Collect

The Application collects and stores the following information **locally** on your device:

**a) Application Password**
- The password you set to protect application access
- Storage location: Device local secure storage (Expo SecureStore)
- Purpose: Verify application access permissions

**b) Browsing History**
- 4chan posts and boards you have browsed
- Browsing timestamps
- Storage location: Device local SQLite database
- Purpose: Provide "History" feature for reviewing previously browsed content

**c) Favorites/Following Data**
- Posts you have favorited
- Users you follow
- Storage location: Device local SQLite database
- Purpose: Provide personalized content management features

**d) Media Cache**
- Image and video files you have browsed
- Storage location: Device local file system
- Purpose: Provide offline browsing and fast loading

**e) Application Settings**
- Your application preferences
- Storage location: Device local storage
- Purpose: Save your personalized settings

### 2.2 Information We Do Not Collect

The Application **DOES NOT** collect:
- Personal identification information (name, email, phone, etc.)
- Device identifiers (IMEI, MAC address, etc.)
- Location information
- Contact lists
- Usage data from other applications
- Any information that can identify you

## 3. Information Use

### 3.1 Local Use

All collected information is used only on your device for:

- **Password Verification**: Protect the application from unauthorized access
- **History**: Display your browsing history
- **Favorites Management**: Manage your favorited and followed content
- **Cache Loading**: Quickly load previously browsed media
- **Settings Preservation**: Remember your application preferences

### 3.2 Not Used For

We **WILL NOT** use your information for:
- Advertising
- User profiling
- Data analytics
- Third-party sharing
- Any commercial purposes

## 4. Information Storage

### 4.1 Storage Location

All data is stored on your device:

- **Secure Storage**: Passwords are encrypted using Expo SecureStore
- **SQLite Database**: History and favorites data stored in local database
- **File System**: Media cache stored in application's private directory

### 4.2 Storage Duration

- Data will remain on your device until you:
  - Manually clear cache
  - Delete history
  - Reset the application
  - Uninstall the application

## 5. Information Sharing

**We do not share your information with any third parties.**

All data is stored only locally on your device. We have no servers, therefore:
- No data is uploaded to the cloud
- No sharing with other users
- No selling or renting to third parties
- No use for any form of data analysis or mining

## 6. Third-Party Services

### 6.1 4chan API

The Application uses 4chan.org's public API to retrieve content. When you use the Application:

- Your device sends requests directly to 4chan.org
- 4chan.org may log your IP address and request information
- These interactions are subject to 4chan.org's own privacy policy
- We have no control over how 4chan.org handles this information

**IMPORTANT NOTE:** We are not an agent of 4chan.org. Your interactions with 4chan.org are direct and do not go through our servers.

### 6.2 Other Third-Party Services

The Application does not use any other third-party services, including but not limited to:
- Analytics services (such as Google Analytics)
- Advertising networks
- Social media integration
- Cloud storage services

## 7. Data Security

### 7.1 Security Measures

We take the following measures to protect your data:

- **Password Encryption**: Passwords are encrypted using device secure storage mechanisms
- **Local Storage**: All data is stored only on your device, not transmitted over network
- **Application Sandbox**: Utilize operating system sandbox mechanisms to isolate application data
- **No Remote Access**: No backdoors or remote access features

### 7.2 User Responsibilities

You are responsible for:
- Protecting your device security (using device lock screen password, etc.)
- Protecting your application password
- Not sharing your device or password with others
- Regularly backing up important data (if needed)

## 8. User Rights

### 8.1 Access and Control

You have complete control over your data:

- **View Data**: View your history, favorites, etc. within the application
- **Delete Data**: 
  - Clear cache (on "Me" page)
  - Delete history
  - Remove favorites
  - Reset entire application
- **Export Data**: Data is stored locally, you can access it through file manager (requires technical knowledge)

### 8.2 Data Portability

Since all data is stored locally:
- Data remains on your device with the application
- Uninstalling the application will delete all data
- We do not provide data export features (because there are no servers)

## 9. Children's Privacy

This Application is not suitable for users under 18. We do not knowingly collect information from children. If you are a parent or guardian and discover your child has used this Application, please uninstall and delete it immediately.

## 10. International Users

This Application can be used globally. Since all data is stored locally:
- No cross-border data transfers involved
- Not subject to specific country/region data protection regulations (such as GDPR, CCPA)
- Your data always remains in your device's location

## 11. Privacy Policy Changes

We may update this Privacy Policy from time to time. For significant changes, we will:
- Display notifications in the application
- Update the "Effective Date" of this document
- Publish update logs in the GitHub repository

Continued use of the Application constitutes acceptance of the updated Privacy Policy.

## 12. Data Breaches

Since the Application does not use servers, there is no traditional "data breach" risk. However:

- If your device is lost or stolen, your local data may be accessed
- We recommend using device lock screen password and application password protection
- Regularly clear sensitive browsing history
